## 2. 수집 항목 (경쟁사 1개당)

### 2-1. 기본 정보 (등록 시 필수)
- 브랜드명
- 자사몰 URL
- 판매 채널 (자사몰 / 29CM / 무신사 / W컨셉 / 스마트스토어 / 쿠팡 / 기타)
- 가격대 (최소~최대)
- 타깃 고객

### 2-2. 상품 정보 (등록 시 선택)
- 베스트셀러 상품명 + 가격 (수동 입력)
- 주요 소재
- 가방 종류 (호보백 / 크로스백 / 토트백 등)

### 2-3. SNS 현황 (등록 시 선택)
- 인스타그램 팔로워 수 (수동 입력)
- 최근 게시물 빈도 (수동)
- 주요 콘텐츠 방식 (룩북 / 숏폼 / 라이브 등)

### 2-4. 불만 리뷰 분석
- 등록 시: 선택 (없어도 등록 가능)
- **AI 분석 실행 시: 필수** — 리뷰 텍스트 없으면 분석 버튼 비활성화
- 입력 방식: 리뷰 텍스트 직접 복붙
- 출력: 주요 불만 카테고리 + 반복 키워드 + 한줄 요약 (Claude API)
- **이력 표시: 최신 분석 결과 펼침 / 이전 이력 날짜별 접기 (아코디언)**

### 2-5. 메모 (등록 시 선택)
- 대표 개인 코멘트
- 위협/기회 판단
- 모니터링 주기 설정

## 3. 주요 기능

1. **경쟁사 추가 폼** — 별도 페이지. 기본 정보(필수) + 나머지(선택) 입력
2. **URL 자동 스크래핑 시도** → 실패 시 수동 입력 fallback (MVP 2 이후, 현재는 부가 기능)
3. **리뷰 분석** — 상세 페이지에서 리뷰 텍스트 입력 후 "AI 분석 실행" → 텍스트 없으면 버튼 비활성
4. **리뷰 분석 이력** — 실행할 때마다 날짜+결과 누적 저장. 최신 결과 펼침 / 이전 이력 아코디언 접기
5. **경쟁사 카드 목록 + 상세 페이지** — 리뷰 미입력 경쟁사는 "분석 대기" 상태 표시
6. **immm 스냅샷 카드 고정 상단** — 자사 데이터를 비교 기준점으로 항상 노출 (카드 형태)
7. **내 브랜드 정보 수정** — immm 카드 우상단 수정 버튼 → 편집 폼 (별도 페이지)
8. **판매 채널별 URL 입력** — 자사몰·29CM·무신사·W컨셉·스마트스토어·쿠팡 각각 URL 등록, 미입력 시 "URL 미입력" 표시
9. **데이터 저장** — 2단계: 1단계 LocalStorage (지금) → 2단계 Vercel 이사 후 AI 리뷰 분석 (4회차)
10. **경쟁사 등록 수** — 10개 기준, 이후 무제한 확장 고려

## 5. 화면 구조

### 화면 1 — 메인 (목록 페이지)
위에서 아래 순서:
1. immm 스냅샷 카드 (고정 최상단, 카드 형태) — 기본 정보 + 상품 라인 6개 + 채널 URL
2. 경쟁사 추가 버튼 — 클릭 시 별도 등록 페이지로 이동
3. 필터 바 — 채널별·분석 상태별 필터
4. 경쟁사 카드 목록 — 브랜드명·가격대·채널·리뷰 분석 상태(완료/대기)

### 화면 2 — 경쟁사 상세 페이지
위에서 아래 순서:
1. 헤더 — 브랜드명, 자사몰 URL, 뒤로가기, 수정/삭제 버튼
2. 기본 정보 섹션
3. 상품 정보 섹션 (미입력 시 "정보 없음 + 수정 버튼")
4. SNS 현황 섹션 (미입력 시 동일)
5. 리뷰 분석 섹션
   - 리뷰 텍스트 입력창
   - "AI 분석 실행" 버튼 (텍스트 없으면 비활성)
   - 최신 분석 결과 (펼침)
   - 이전 분석 이력 (날짜별 아코디언, 기본 접힘)
6. 메모 섹션

### 화면 3 — 경쟁사 등록 폼 (별도 페이지)
위에서 아래 순서:
1. 브랜드명 (필수)
2. 자사몰 URL (필수)
3. 판매 채널 (필수, 다중 선택)
4. 가격대 (필수)
5. 타깃 고객 (필수)
6. 상품 정보 (선택)
7. SNS 현황 (선택)
8. 메모 (선택)
9. 저장 버튼

### 화면 4 — 내 브랜드(immm) 편집 폼 (별도 페이지)
위에서 아래 순서:
1. 기본 정보 — 브랜드명·슬로건·가격대·타깃·소재·생산·인스타 팔로워 (필수 항목 포함)
2. 판매 채널별 URL — 자사몰·29CM·무신사·W컨셉·스마트스토어·쿠팡 (선택)
3. 상품 라인 설명 — im lovely~im glossy 6개 각각 설명 입력 (선택)
4. 저장 버튼

## 7. 크롤링 리스크 (명확히 인지)

> 무신사·29CM은 봇 차단(403/429) 강함. Vercel 배포 환경에서 Playwright 실행 제한. Cafe24 기반 자사몰은 브랜드마다 HTML 구조가 달라 범용 파서 불안정. 현실적으로 자동 수집 성공률 50% 미만 예상.
>
> **대응 방향**: 크롤링 실패 시 수동 입력 fallback 구조를 먼저 완성. 크롤링은 "되면 좋고 안 돼도 무방한" 부가 기능으로 취급.
